§ I

What is a Turing Machine?

A Turing machine is a mathematical model of computation — not a physical object, but an idea. It imagines a device with an infinite paper tape, a read/write head that can move left or right, and a small set of rules (a finite state control) that decides what to do next.

That minimal setup is enough to compute anything a modern computer can compute. This is the Church–Turing thesis, and it is the reason this simple abstraction underlies every programming language, operating system, and algorithm you will ever study.

“It is possible to invent a single machine which can be used to compute any computable sequence.” — A. M. Turing, On Computable Numbers, 1936

§ III

The Formal Seven-Tuple

A deterministic Turing machine is formally defined as a 7-tuple. Every example in this simulator is one of these objects.

M = ( Q , Σ , Γ , δ , q0 , qaccept , qreject )
Q
Finite set of states — every place the control can be in, such as q0, q1.
Σ (Sigma)
Input alphabet — the symbols a user is allowed to feed the machine. Does not include the blank.
Γ (Gamma)
Tape alphabet — everything the tape may contain, including the blank _ and any auxiliary marks.
δ (delta)
Transition function — the rulebook: δ : Q × Γ → Q × Γ × {L, R}.
q0
Start state — where the machine begins execution.
qaccept
Accept state — halts and the input is said to be accepted.
qreject
Reject state — halts and the input is rejected.
§ IV

The Four Working Parts

i

The Tape

An infinite sequence of cells, extending both directions. Each cell holds one symbol from the tape alphabet Γ, or a blank. This is both the machine's input and its memory.

ii

The Head

A single reader/writer that sits over exactly one cell. On each step it can read, overwrite that cell, then shift one position left or right. The amber triangle in the simulator shows where it is.

iii

The Control

A finite set of states that remembers what the machine is "doing right now". It is not memory in the usual sense — it is a tiny brain that decides the next move based on the current state and the symbol being read.

iv

The Rules (δ)

A lookup table. Given (current state, read symbol) it returns (new state, symbol to write, direction to move). This is the entire program the machine knows.

§ V

One Cycle of Execution

  1. Read the symbol under the head

    The head observes exactly one cell of the tape. Whatever symbol lives there is the input to the rule lookup.

  2. Consult the transition function δ

    Using (current state, symbol read) as a key, find the matching rule. If no rule exists for that pair, the machine halts and rejects.

  3. Write the new symbol

    Replace the symbol in the current cell with the one the rule specifies. Often the machine writes the same symbol back — which is equivalent to leaving it alone.

  4. Change the state

    Update the control to the next state indicated by the rule. This is how the machine remembers what phase of the computation it is in.

  5. Move the head one cell

    Shift exactly one position left (L) or right (R). The head never stays still and never jumps — single steps only.

  6. Loop — or halt

    If the new state is q_accept or q_reject, the machine stops. Otherwise, return to step one and repeat. A machine may also loop forever, which is why some problems are undecidable.

§ VI

How to Present This to Your Class

A suggested script for a three-to-five minute demo. Works equally well for a classmate or a professor.

  1. Begin with the big idea

    "A Turing machine is the simplest device that can compute anything a computer can. Everything on my laptop — from a browser to an AI model — is, in theory, just a very fast Turing machine."

  2. Introduce the parts on screen

    Point to the tape, the head (amber triangle), the state box, and the transition table. Say: "These four things — and only these four things — run the whole computation."

  3. Pick a simple example first

    Start with Binary Increment and input 1011. Predict the output (1100). Press Step a few times. Read the execution log aloud: "It moved right because the rule δ(q0, 1) = (q0, 1, R) said so."

  4. Emphasise the transition function

    Point at the highlighted row in the transition table as it fires. This is the moment the audience understands that "programming" a Turing machine just means writing this table.

  5. Show acceptance and rejection

    Switch to a^n b^n Recognizer. Load aaabbb — it accepts. Then try aabbb — it rejects. This is the clearest demonstration of a machine deciding a language.

  6. Close with the Church–Turing thesis

    "Every algorithm that can possibly be computed can be computed by a machine like this. That is why we study it — this is the definition of computation itself."
